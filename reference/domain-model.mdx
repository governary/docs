# Domain Model

These are the core data objects in the Governix MVP.

## Core objects

| Object | Purpose |
| --- | --- |
| `tenant` | Governance, quota, showback, and audit boundary |
| `application` | Ingestion unit under a tenant; API key and usage attribution |
| `tenant_policy` | KB and model access rules for a tenant |
| `tenant_quota` | Monthly limits and threshold behavior |
| `request_ledger` | Per-request event chain |
| `usage_daily` | Daily aggregated usage per tenant or application |
| `policy_evaluation_logs` | Policy match result per request |
| `audit_exports` | Export job tracking |
| `users` | Console admin users only |

## Roles

| Role | Permissions |
| --- | --- |
| `admin` | Full read and write |
| `operator` | CRUD on tenants, apps, policies, quotas, and ledger |
| `viewer` | Read-only |

## Security rules

- All management API routes require an authenticated console session
- Runtime routes require an application API key
- Application API keys are stored as hashes only
- Plaintext keys are returned once on create or rotate
- Runtime API keys are only for Governix runtime routes, not for direct AWS access
- Governix stores summaries and metadata, not full prompt or response bodies
