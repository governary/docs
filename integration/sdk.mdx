---
title: "Integrate Governix with your Bedrock app using the SDK"
description: "Add Governix policy evaluation and event ingestion to your Bedrock app with the SDK — no proxy gateway or infrastructure changes required."
---

Governix provides a Python SDK that lets you add tenant-aware policy enforcement and usage tracking to any existing AWS Bedrock application. This page walks you through installing the SDK, authenticating, evaluating policy before each Bedrock call, and sending retrieval and generation events after the call completes.

<Note>
  Governix works as a control plane alongside your existing Bedrock integration. No proxy gateway is required — you keep your current Bedrock calls and add Governix before and after them.
</Note>

<Steps>
  <Step title="Install the SDK">
    Install the Governix SDK using pip.

    ```bash
    pip install governix
    ```
  </Step>

  <Step title="Authenticate with your API key">
    Each application in Governix has an API key used for runtime attribution. Retrieve your key from the [app.governary.io](https://app.governary.io) admin console, then pass it when initializing the client.

    ```python
    from governix import GovernixClient

    client = GovernixClient(api_key="gvx_your_application_api_key")
    ```

    Keep your API key in an environment variable rather than hardcoding it in source code.

    ```python
    import os
    from governix import GovernixClient

    client = GovernixClient(api_key=os.environ["GOVERNIX_API_KEY"])
    ```
  </Step>

  <Step title="Evaluate policy before the Bedrock call">
    Before making any Bedrock request, call the Governix policy evaluation API. Pass the tenant ID, application ID, the requested knowledge base, and the requested model. Governix returns a decision you must act on before proceeding.

    ```python
    decision = client.evaluate_policy(
        tenant_id="tenant_acme",
        app_id="app_rag_assistant",
        knowledge_base_id="kb_product_docs",
        model_id="anthropic.claude-3-sonnet-20240229-v1:0",
    )
    ```

    The `decision` object contains a `status` field with one of the following values:

    | Status | Meaning |
    |---|---|
    | `allow` | Proceed with the Bedrock call as requested. |
    | `deny` | Do not proceed. Return an error to the caller. |
    | `quota_block` | Tenant has exceeded their usage quota. Do not proceed. |
    | `force_filter` | Proceed, but apply the filter returned in `decision.filter`. |
    | `downgrade_model` | Proceed, but use `decision.model_id` instead of the requested model. |
  </Step>

  <Step title="Handle the policy decision">
    Branch on the decision status before making the Bedrock call.

    ```python
    if decision.status in ("deny", "quota_block"):
        raise PermissionError(f"Request blocked by Governix policy: {decision.status}")

    # Apply overrides if required
    kb_filter = decision.filter if decision.status == "force_filter" else None
    model_id = decision.model_id if decision.status == "downgrade_model" else "anthropic.claude-3-sonnet-20240229-v1:0"

    # Proceed with the Bedrock call
    bedrock_response = call_bedrock(
        knowledge_base_id="kb_product_docs",
        model_id=model_id,
        query="How do I reset my password?",
        filter=kb_filter,
    )
    ```
  </Step>

  <Step title="Send events after the Bedrock call">
    After the Bedrock response is returned, send a retrieval event and a generation event to Governix. These events update quota balances and write entries to the audit ledger.

    <CodeGroup>

    ```python retrieval event
    client.ingest_retrieval_event(
        tenant_id="tenant_acme",
        app_id="app_rag_assistant",
        knowledge_base_id="kb_product_docs",
        filters_applied=kb_filter,
        chunks_retrieved=bedrock_response.retrieved_chunks,
    )
    ```

    ```python generation event
    client.ingest_generation_event(
        tenant_id="tenant_acme",
        app_id="app_rag_assistant",
        model_id=model_id,
        input_tokens=bedrock_response.usage.input_tokens,
        output_tokens=bedrock_response.usage.output_tokens,
        estimated_cost_usd=bedrock_response.usage.estimated_cost_usd,
    )
    ```

    </CodeGroup>

    <Warning>
      Send events even when the Bedrock response contains an error. Partial usage is still recorded and contributes to quota consumption.
    </Warning>
  </Step>
</Steps>

## Complete example

The following snippet shows the full SDK integration in a single function.

```python
import os
import boto3
from governix import GovernixClient

client = GovernixClient(api_key=os.environ["GOVERNIX_API_KEY"])

def handle_request(tenant_id: str, query: str) -> str:
    app_id = "app_rag_assistant"
    knowledge_base_id = "kb_product_docs"
    requested_model_id = "anthropic.claude-3-sonnet-20240229-v1:0"

    # Step 1: Evaluate policy
    decision = client.evaluate_policy(
        tenant_id=tenant_id,
        app_id=app_id,
        knowledge_base_id=knowledge_base_id,
        model_id=requested_model_id,
    )

    if decision.status in ("deny", "quota_block"):
        raise PermissionError(f"Request blocked: {decision.status}")

    kb_filter = decision.filter if decision.status == "force_filter" else None
    model_id = decision.model_id if decision.status == "downgrade_model" else requested_model_id

    # Step 2: Call Bedrock
    bedrock = boto3.client("bedrock-agent-runtime")
    response = bedrock.retrieve_and_generate(
        input={"text": query},
        retrieveAndGenerateConfiguration={
            "type": "KNOWLEDGE_BASE",
            "knowledgeBaseConfiguration": {
                "knowledgeBaseId": knowledge_base_id,
                "modelArn": model_id,
                "retrievalConfiguration": {
                    "vectorSearchConfiguration": {"filter": kb_filter}
                } if kb_filter else {},
            },
        },
    )

    # Step 3: Send events
    client.ingest_retrieval_event(
        tenant_id=tenant_id,
        app_id=app_id,
        knowledge_base_id=knowledge_base_id,
        filters_applied=kb_filter,
        chunks_retrieved=response["citations"],
    )
    client.ingest_generation_event(
        tenant_id=tenant_id,
        app_id=app_id,
        model_id=model_id,
        input_tokens=response["usage"]["inputTokens"],
        output_tokens=response["usage"]["outputTokens"],
        estimated_cost_usd=response["usage"]["estimatedCostUsd"],
    )

    return response["output"]["text"]
```
