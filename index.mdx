# Governix

Governix is the open-source, tenant-aware control, metering, and audit layer for Bedrock-powered AI.

Governix is a control plane for SaaS teams building multi-tenant AI and RAG products on AWS Bedrock. It does not replace Bedrock, vector databases, or your application. Instead, it adds governance and observability around them.

## What Governix adds

- Tenant-aware request policy before a Bedrock call
- Per-tenant usage, quota, and showback visibility
- Audit-grade retrieval and generation ledger records

## Core model

Governix is a sidecar control plane, not an inline proxy.

1. Your application calls Governix before a Bedrock request.
2. Governix returns a policy and quota decision.
3. Your application enforces that decision and calls Bedrock directly.
4. Your application sends runtime event metadata back to Governix.

## MVP scope

Included in MVP:

- Tenant CRUD
- Application CRUD and API key rotation
- Tenant-scoped policy CRUD and policy testing
- Tenant quota configuration
- Runtime policy evaluation API
- Runtime event ingestion API
- Usage aggregation and showback export
- Request ledger UI and async CSV export
- Basic dashboard and alert visibility

Out of scope in MVP:

- Inline Bedrock proxy or gateway
- Multi-cloud support
- Full billing and payments
- Application-scoped policy
- Prompt playgrounds and eval platforms
- Enterprise SSO, SCIM, and org hierarchy

## Tech stack

- Frontend: Next.js App Router and TypeScript
- UI: shadcn/ui and Tailwind CSS
- Backend API: Next.js Route Handlers
- Database: PostgreSQL
- ORM: Drizzle ORM
- Auth: Email and password with secure cookies
- Runtime SDK: TypeScript
- Testing: TDD and Playwright Test
- Local development: Docker Compose
